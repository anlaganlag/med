# 病历质量评估NLP流程图

```mermaid
graph TD
    A[非结构化病历文本] --> B[数据预处理]
    B --> B1[文本清洗与分词]
    B1 --> B2[停用词过滤]
    B2 --> B3[文本标准化]
    B3 --> C[自然语言处理 (NLP)]
    
    C --> C1[医学术语识别与标准化]
    C1 --> C2[时序关系分析]
    C2 --> C3[上下文关系理解]
    C3 --> D{实体提取与规范化}
    
    D --> E[症状提取]
    D --> F[体征提取]
    D --> G[化验结果提取]
    D --> D1[用药信息提取]
    D --> D2[手术操作提取]
    D --> D3[透析相关参数提取]
    
    E --> E1[症状严重程度评估]
    F --> F1[体征异常判断]
    G --> G1[化验结果异常分析]
    D1 --> D1A[药物剂量合规性]
    D2 --> D2A[操作规范性评估]
    D3 --> D3A[透析参数合理性]
    
    E1 --> H[结构化数据]
    F1 --> H
    G1 --> H
    D1A --> H
    D2A --> H
    D3A --> H
    
    H --> H1[实体关系提取与分析]
    H1 --> H2[医学知识图谱匹配]
    H2 --> I[规则匹配与缺口识别]
    
    I --> I1[性能指标评估]
    I1 --> I1A[准确率评估]
    I1 --> I1B[召回率评估]
    I1 --> I1C[F1分数计算]
    I1A --> J{缺口类型}
    I1B --> J
    I1C --> J
    
    J --> K[格式规范缺口<br>如未记录姓名]
    J --> L[内容完整性缺口<br>如未记录透析器型号]
    J --> M[医学准确性缺口<br>如超滤量与体重不符]
    J --> N[专科要求缺口<br>如内瘘评估笼统]
    J --> J1[时序逻辑缺口<br>如治疗时间顺序错误]
    J --> J2[关联性缺口<br>如症状与诊断不匹配]
    
    K --> O[整改建议生成]
    L --> O
    M --> O
    N --> O
    J1 --> O
    J2 --> O
    
    O --> O1[建议优先级排序]
    O1 --> O2[建议可行性评估]
    O2 --> P[指导意见输出<br>如补充透析器型号]
    
    P --> P1[意见解释性生成]
    P1 --> P2[参考文献与指南关联]
    P2 --> Q[临床决策支持系统 (CDSS)]
    
    Q --> Q1[医学知识库更新]
    Q1 --> R[医生决策与调整]
    
    R --> R1[反馈学习机制]
    R1 --> S[更新病历]
    
    S --> S1[病历质量评分]
    S1 --> A
```

## 流程说明

此流程图展示了使用自然语言处理技术对非结构化病历文本进行质量评估的完整流程，从文本预处理到最终的质量评分和反馈机制。
